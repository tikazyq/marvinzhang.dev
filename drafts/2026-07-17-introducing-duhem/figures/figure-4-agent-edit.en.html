Can you read a diff and tell if the promise moved?

Hand an agent the same two changes — one implementation, one requirement. The difference isn’t line count; it’s whether, after the edit, you can see if what you promised moved.

The difference isn’t how many lines change — it’s whether a diff tells you if what you promised moved. A VD does: look at whether it touched a criterion (the promise) or a check (how it’s verified). In imperative tests every change looks the same — so the agent’s most dangerous move, quietly weakening a check to go green, has nowhere to be caught.
