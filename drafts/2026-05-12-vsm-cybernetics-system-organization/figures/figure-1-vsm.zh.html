Figure 01 — 可生存系统模型

五个层级，一条侧通道，一条警报通路

每一层都活在某处。左边是它在传统工程团队里历史上的角色，右边是在跑 AI 编程智能体的流水线里的对应。
痛快通道（警报路径）—— S1 → S5，绕过 S2 / S3 / S4
S3* —— 侧通道（与 S3 平行，不是下属）
S5
政策
传统团队
创始人发言、公司价值观、高管 All-Hands 反复强调的几个要点。
智能体流水线
AGENTS.md、CLAUDE.md、原则文档、ADR。
S4
情报
传统团队
产品、用户研究、销售情报、客户成功。
智能体流水线
几乎总是不存在的那部分。
S3
控制
传统团队
Scrum Master、工程经理、迭代规划、产能分配。
智能体流水线
DAG 计划器、工作分派策略、会话分诊。
S3* 审计 —— 侧通道
传统团队 SRE、QA、安全审计——在工程汇报链之外。
智能体流水线 对路由系统 / 计划器 / 监督逻辑的审计，独立于运行它们的人。
S2
协调
传统团队
分支策略、PR 评审队列、结对编程协议。
智能体流水线
Git worktree、沙盒化运行时、共享写路径的锁。
S1
作业
传统团队
写代码的开发者、跑测试的进程、执行部署的脚本。
智能体流水线
一次智能体会话——一次 coding agent 调用，交付一个合并请求。
