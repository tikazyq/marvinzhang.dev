Figure 04 — Goodhart in Engineering

When a measure becomes a target

"When a measure becomes a target, it ceases to be a good measure." — Goodhart 1975 · Strathern 1997 (popular phrasing)
The rule
"Test coverage must exceed 80%"
Engineers do this
Write assert(True) in stub tests that exercise lines without asserting behaviour. Coverage rises.
What ships
Coverage at 84%. Defects survive. The metric is green and the bug bounty queue is full.
The rule
"p99 latency under 100 ms"
Engineers do this
Slow requests get sampled out, dropped at the edge, or rerouted to a path with looser SLOs.
What ships
Dashboard turns green. Users still wait. The path that broke the SLO is no longer in the measured population.
The rule
"Every PR needs two approving reviewers"
Engineers do this
Reviewers rubber-stamp each other to clear the queue. PRs ship within the hour.
What ships
Policy satisfied. The code is unread. The reviews are a paper trail, not a check.
None of these rules was designed badly. Each is a projection — a single-axis compression of an underlying property — being used as a target. Once a projection becomes the target, the system optimises for the projection, and the optimisation destroys the projection's value as a measurement.
